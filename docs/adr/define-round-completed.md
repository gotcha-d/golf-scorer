# Title： ラウンドを終了したかどうかをドメインオブジェクトとするか議論

## コンテキスト(Context)
### なぜこの決定が必要だったか
ゴルフスコア算出アプリにおいて、18ホール分のスコアが入力された時点でグロススコアの集計やハンディキャップ計算のなどの処理を行う必要がある。
この「全ホール入力が完了した瞬間」は、「ヒト・モノ・コト」で分類するとコトに該当する。

## Decision
Round集約が全ホールのスコア入力を検知した時点で、ラウンド終了というドメインイベントを発火する。
このイベントを受け取ったハンドラ（スコア計算、ハンディキャップ計算）がそれぞれの処理を担当する。
- ドメインイベントクラス名: RoundCompleted
  - ドメインイベントは、ドメインにおいて重要なことが発生されたことをきっかけに発火するので過去形にする
- 発火タイミング: ホール番号で区別される全ホールのスコアが入力されたタイミング
- 発火後の永続化と順序については別途検討する。

## Consequences
- Roundは後続処理の具体的な処理内容を知らない
- 新しい処理(通知・・PDF出力など)追加する際、Roundへの変更が発生しない。
  - 新しい処理がRoundCompletedというイベントを検知して処理を行うだけ
- これは、SOLID原則のうち「依存性の逆転」の法則に基づいている。
